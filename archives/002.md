# TASK-002: Build Agent Work Dashboard

**Status:** Pending
**Requested By:** Human
**Priority:** High

## 🎯 Objective
Create a professional, data-driven UI that serves as the "Command Center" for this monorepo. This dashboard must visualize the work history stored in `data/tasks.json`.

## 📋 Requirements

### 1. Frontend Development
- **Location:** `apps/dashboard` (React/Next.js).
- **Styling:** Use **Tailwind CSS**. Implement a dark-themed, "Matrix-style" or "Terminal" aesthetic (Slate-900 backgrounds with Emerald-500 accents).
- **Components:**
    - **Header:** Title "Agentic Monorepo Monitor" and a live-updating clock.
    - **Stats Row:** Summary cards showing "Total Tasks", "Completed", and "Pending".
    - **Task Feed:** A clean list or table displaying every entry from `data/tasks.json`.

### 2. Data Integration
- **Source of Truth:** The app must read directly from `data/tasks.json`.
- **Fields to Display:**
    - `id`: The Task ID.
    - `title`: Short description.
    - `status`: Visual indicator (e.g., a green dot for "Completed").
    - `timestamp`: The date/time the agent finished the work.

### 3. E2E Validation
- **Test Suite:** Create `apps/dashboard/tests/dashboard.spec.ts` using **Playwright**.
- **Assertions:** - Verify the dashboard loads successfully.
    - Verify that at least one task entry from the JSON file is rendered in the UI.

- **Responsiveness:** The dashboard must be readable on mobile and desktop.
