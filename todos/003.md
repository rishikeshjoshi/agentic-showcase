## 🎯 Objective
Decouple the Dashboard UI from the direct file system by creating a Node.js/Express (or Python/FastAPI) backend. This server will serve as the official API for task data.

## 📋 Requirements

### 1. Backend Development
- **Location:** `services/api-server`.
- **Framework:** Node.js with Express (preferred).
- **Endpoint:** `GET /api/tasks`
    - **Logic:** Read `data/tasks.json` from the root and return the array as JSON.
    - **Error Handling:** Return a 500 error if the file is missing or unreadable.
- **CORS:** Enable Cross-Origin Resource Sharing so the Dashboard app can fetch data.

### 2. Dashboard Integration
- **Refactor:** Update the React app in `apps/dashboard` to stop importing the JSON file directly.
- **Fetch:** Use `useEffect` or `react-query` to fetch data from `http://localhost:port/api/tasks`.

### 3. E2E Validation
- **Test Suite:** `services/api-server/tests/api.spec.ts`.
- **Assertions:** - Verify that a GET request to `/api/tasks` returns a 200 status code.
    - Verify the response body is an array containing at least one task.

---

## 🛡️ Guardrail Compliance
- **Performance:** Implement basic caching or ensure file reading is non-blocking (async/await).
- **Technology:** Use a standard port (e.g., 3001) and document it in a local `README.md` within the service folder.
